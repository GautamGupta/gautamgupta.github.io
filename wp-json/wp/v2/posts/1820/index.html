You\u00a0are a mobile developer with an app on the store, serving thousands of people in different languages. But localizing the app \u2014 back and forth with your in-house translators every time you release a build \u2014 is a pain. Here is how to fix that, with a relatively undocumented yet open source software called GlotPress.<\/p>\n

Goal:\u00a0<\/strong>Host an internal website where translators can login.\u00a0Import\u00a0strings from your iOS and Android builds.\u00a0Export\u00a0back the translated strings. Preview<\/a>.<\/p>\n

Setup<\/h4>\n
    \n
  1. GlotPress is written in PHP and uses MySQL. Setup the LAMP stack<\/a>, create a database<\/a>\u00a0and in htdocs<\/em> (or equivalent):\n
    svn co\u00a0http:\/\/svn.glotpress.org\/trunk\/ .<\/pre>\n<\/li>\n
  2. Copy gp-config-sample.php<\/em> located in the root directory to gp-config.php<\/em>. Edit the file to enter in your database configuration in\u00a0GPDB_*\u00a0<\/em>defines. Also go here<\/a>\u00a0and replace the *_KEY<\/em> values.<\/li>\n
  3. Run the installer by going to the domain pointing to your server.\u00a0Login.\n
    \u00a0<\/div>\n<\/li>\n
  4. Create a new project named Android<\/em>. Expand Project Actions<\/em> and click Import Originals<\/em>. Upload the strings.xml<\/em> file from your Android project.
    \n<\/li>\n
  5. Go back to the project. You would not see anything yet because you have just added the originals yet, not Translation Sets (ie. other languages). Expand Project Actions,<\/em>\u00a0click\u00a0New Translation Set<\/em> and add a locale.
    \n<\/li>\n
  6. If you have existing translations for the locale, scroll down and click Import Translations<\/em>.
    \n<\/li>\n
  7. Adding the iOS project: Follow the same procedure as in step 4, but this time select the\u00a0Mass-create Translation Sets<\/em> option to import the settings from the Android project. I will leave rest of the nifty automated processes that GlotPress offers for you to discover.<\/li>\n
  8. Google Translation (nice to have): This shows a small\u00a0“Translation from Google” link below the translation box. It can\u00a0fetch an auto-generated translation from Google that the human translator can correct. However it requires a (paid) Google Translate API key<\/a>. To enable it, you need to edit gp-config.php<\/em> and add this:\n
    define('GP_GOOGLE_TRANSLATE_KEY', '<API KEY>');<\/pre>\n<\/li>\n
  9. Exporting: Go to the locale to export translations from, select Android XML and hit export.<\/li>\n
  10. Adding translators: There\u00a0is no user\u00a0management GUI in GlotPress yet. More on this below. To add a translator, copy this script<\/a> into scripts\/add-new-user.php<\/em>. Then run this command from the htdocs<\/em> directory:\n
    .\/scripts\/add-new-user.php\u00a0-u <username> -e <email> -p <password><\/pre>\n<\/li>\n<\/ol>\n
    Further<\/h4>\n
      \n
    • For Android developers: If you are new to localization, read Google’s documentation on managing strings<\/a>. You can also set some strings to be not translated<\/a> if you use strings.xml<\/em> to also store API keys etc.<\/li>\n
    • Automated Import\/Export from App’s Git Repository (because automated translation handling isn’t enough)<\/em>: I had developed a tool<\/a> for LimeSurvey about 3 years ago\u00a0that would run as a cron, automatically add new strings into GlotPress and commit<\/a> new translations.\u00a0To make it work with mobile apps would take some effort.<\/li>\n
    • Structuring Projects: Check out how<\/a> WordPress (parent project of GlotPress) uses it.<\/li>\n
    • Handbook for translators: WordPress also has a small tutorial<\/a> you can link your translators to.<\/li>\n
    • Why there is no user management: GlotPress has been in existence for 7 years<\/a>, but it has received relatively less attention. I think it’s only WordPress.org that uses GlotPress extensively, and since one can replace GlotPress user tables with WordPress’, no work was probably done to add user management into GlotPress itself. It will probably be made into a WordPress plugin soon. Update: Until then, this seems to be worth a try<\/a>.<\/li>\n<\/ul>\n","protected":false},"excerpt":{"rendered":"
