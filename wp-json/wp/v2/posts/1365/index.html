I was actively working with LimeSurvey during Google Code-in ’11<\/a> and had promised them to integrate GlotPress<\/a> with LimeSurvey (user tables) to help the translators and completed it some weeks back (though this was not the end result that they actually wanted, I’ll work on that later – thought I should write this before I forget the process myself – I know I have a short memory ;)).<\/p>\n

Integrating GlotPress & LimeSurvey<\/h3>\n

Assuming you already have a LimeSurvey install, whose user table you want to use.<\/p>\n

    \n
  1. svn checkout\u00a0http:\/\/svn.glotpress.<\/wbr>org\/trunk\/<\/a>\u00a0glotpress<\/li>\n
  2. Download<\/a> the three custom files found in the root folder (or clone the repository<\/a>) and put them in the glotpress folder.<\/li>\n
  3. Edit gp-config.php<\/em> for the database details, you can also regenerate the salts on your wish. Scroll down below to check if\u00a0CUSTOM_USER_TABLE<\/em> constant shows the correct user table name (check the table prefix lime_<\/em>).<\/li>\n
  4. Open the folder location in the browser to run the GlotPress installation process. Write the generated .htaccess<\/em>.<\/li>\n<\/ol>\n

    Now do any of the two steps:<\/p>\n

      \n
    1. Make all<\/span> superadmins in LimeSurvey administrators in GlotPress – put the plugins\/permission.php<\/em>\u00a0file\u00a0in plugins<\/em> folder.<\/li>\n
    2. Make only some<\/span> people in LimeSurvey administrators in GlotPress – run this query –
      \nINSERT INTO `gp_permissions` (`id` ,`user_id` ,`action` ,`object_type` ,`object_id`) VALUES (NULL , ‘1’, ‘admin’, NULL , NULL);<\/em>
      \n(assuming gp_<\/em> is the prefix you’ve chosen) where the corresponding `user_id`<\/em> is the user id in lime_users<\/em> table whom you want to make the admin in GlotPress.<\/li>\n<\/ol>\n

      And, done! No core files modified.\u00a0Now you can login with that id, create new projects, and add other users as translation approvers, etc. Normal users can only suggest translations.<\/p>\n

      Integrating GlotPress & Any Software<\/h3>\n

      Now that I have written about how to integrate GlotPress with LimeSurvey, I also wanted to write about how you can actually integrate GlotPress with any other software which has a user table which GlotPress could use.\u00a0There are basically four modifications you need to make.\u00a0These modifications to the original classes were done on GlotPress r683.\u00a0Refer<\/strong> to this\u00a0commit<\/a>\u00a0to see how the core files were modified to suit the needs, explanation below.<\/p>\n

      User class<\/strong>\u00a0(class.ls-users.php<\/em>)<\/h4>\n

      You need to make GlotPress aware of your user table by editing the WP_Users<\/em> class.<\/p>\n

        \n
      1. Find\/replace all the user table column names with yours. Also edit the _put_user()\u00a0method<\/a>.<\/li>\n
      2. Disable<\/a>\u00a0(and here<\/a>) search by user_nicename<\/em>\u00a0if no such column exists and you can do that with username instead\u00a0(similarly with other columns).<\/li>\n
      3. Correct<\/a> set_password()<\/em><\/li>\n
      4. Make<\/a> appropriate mapping\/addition of values<\/li>\n<\/ol>\n

        Password Hashing (class.ls-pass.php<\/em>)<\/h4>\n

        It is likely that your software would be using a password hashing algorithm other than PHPass, that is used by GlotPress. If not, you can skip this step but if that’s the case, you’d need to edit the hash_password()<\/em>\u00a0method<\/a> in WP_Pass<\/em>\u00a0class with your hashing algorithm.<\/p>\n

        Making GlotPress aware of the changes (gp-config.php<\/em>)<\/h4>\n

        Refer the lines 54-60<\/a>\u00a0–\u00a0you need to tell the GlotPress to not use its own user table and user\/hashing classes and instead use yours.<\/p>\n

        Permissions (plugins\/permission.php)<\/em><\/h4>\n

        GlotPress uses a gp_permissions<\/em>\u00a0table than the user meta table to store permissions, making our work easier.\u00a0As described above, you have two options for permissions – you can either run the sql query to convert a particular user into an administrator or if your user table contains a column which differentiates the administrators and you can use that, you can just edit and activate this plugin<\/a>.<\/p>\n

        Bonus<\/em>\u00a0– Plugins!<\/h3>\n

        I have also created a Github repository<\/a> with four plugins which my friend Ben L.<\/a> had written that you\u00a0may\u00a0find useful – just put them in the plugins folder and comment\/uncomment the last line to activate\/deactivate them.<\/p>\n
