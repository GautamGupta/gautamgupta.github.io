I am glad to announce the release of Plugin Browser for bbPress<\/em><\/a> v0.2! This plugin provides an interface in the bbPress admin area where a keymaster can install, uninstall, activate, deactivate and upgrade plugins that are featured in the bbPress plugin repository<\/a>. It has been 3 years since the plugin was developed by Sam Bauers, and now he has made me a committer of the plugin, for which I am thankful to him.<\/p>\n

Changelog<\/h3>\n
    \n
  • Compatibility with bbPress 1.0<\/li>\n
  • Usage of reliable WP_Http instead of cURL or fopen<\/li>\n
  • Automatically gets the new repository list if available, when you go to the plugin browser page<\/li>\n
  • The directory folder of a plugin now need not have pb–<\/em> in its name, nor a pb–revision.php<\/em>, which means that now the plugins are checked with their version numbers.<\/li>\n
  • Added i18n support<\/li>\n
  • Coding cleanups (including WordPress style coding), optimizations and improvements<\/li>\n<\/ul>\n

    Todo<\/h3>\n
      \n
    • Add a daily auto-check for plugins and notify the administrator if any plugin has a new version<\/li>\n<\/ul>\n

      Notes<\/h3>\n
        \n
      • If you were using an older version of plugin browser for bbPress than v0.2, then it is recommended that you remove pb--<\/code> from the plugins’ directory name in the my-plugins<\/code> folder.<\/li>\n
      • If the plugin list is not updating for some reason, then go to the plugin browser page, and in the URL add this – &force_update=1<\/code> and press enter. The plugin should then make a new list of the available plugins.<\/li>\n<\/ul>\n
